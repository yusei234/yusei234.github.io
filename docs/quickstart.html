Quick Start Guide

Get up and running with VFF-Nav in minutes

Prerequisites

  • Ubuntu 22.04+ or 24.04+
  • ROS 2 Jazzy Jalisco
  • Gazebo Fortress or Garden
  • PX4 Autopilot
  • Python 3.10

Installation

1. Clone the repository

git clone https://github.com/miguel-usp/vff-nav.git
cd vff-nav

2. Setup the simulation environment

# Install ROS 2 dependencies
sudo apt update
sudo apt install ros-jazzy-desktop

# Setup PX4 environment
git submodule update --init --recursive
make px4_sitl gazebo-classic

3. Build the workspace

colcon build
source install/setup.bash

Running the Simulation

1. Launch the simulation

ros2 launch vff_nav simulation.launch.py

2. Run the VFF algorithm

In another terminal:

ros2 run vff_nav vff_controller

3. Monitor the simulation

# View robot transformation
ros2 run rqt_tf_tree rqt_tf_tree

# Monitor topics
ros2 topic list
ros2 topic echo /drone/position

Configuration

Customize the VFF parameters in the configuration file:

# config/vff_params.yaml
goal_charge: -1.0
goal_constant_force: 2.0
obstacle_saturation: 5.0
damping_constant: 0.5

Troubleshooting

Common Issues

  • Gazebo not launching: Ensure all ROS 2 environment variables are set
  • PX4 connection issues: Check MAVLink communication ports
  • Permission denied: Add your user to the dialout group: sudo usermod -a -G dialout $USER

Getting Help

If you encounter issues, please:

  1. Check the GitHub Issues

  3. Create a new issue with detailed error information
